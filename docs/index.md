## Custom Project

### Dataset

The Kafka producer `kafka_producer_case.py` uses the unmodified `sales.csv` dataset.

This dataset contains simulated sales transaction records.

Each record represents a single customer purchase.

Fields in the dataset include:

- `order_id`: unique identifier for the transaction
- `datetime`: timestamp for the purchase event
- `region_id`: geographic region
- `currency_code`: currency used
- `product_id`: purchased product ID
- `unit_price`: price per unit
- `quantity`: number of units purchased
- `is_online`: (T/F) whether the purchase was online
- `customer_id`: unique customer identifier
- `is_new_customer`: whether the customer is new
- `device_type`: device used
- `payment_method`: payment type
- `referral_source`: traffic source
- `discount_code`: optional discount code
- `customer_note`: optional customer comment

Several reference datasets were used by the consumer for validation and enrichment, including:

- `regions.csv` for tax rates and region validation
- `products.csv` for valid product IDs
- `currencies.csv` for allowed currency codes
- `discount_codes.csv`

### Data Contract

The data contract defines what constitutes a valid sales message.

Required fields:

- `order_id`
- `datetime`
- `region_id`
- `currency_code`
- `product_id`
- `unit_price`
- `quantity`
- `is_online`
- `customer_id`
- `payment_method`

Optional fields:

- `is_new_customer`
- `device_type`
- `referral_source`
- `discount_code`
- `customer_note`

Fields that must match allowed values or reference data:

- `region_id`: must exist in the regions reference table
- `product_id`: must exist in the products reference table
- `device_type`, `payment_method`, `referral_source`, and `currency_code`
  must be within defined allowed sets
- `quantity`: must be a positive integer
- `datetime`: must be a valid timestamp
- boolean fields must contain valid boolean-like text

A message is considered valid if all required fields are present and all validation rules pass.

Messages are marked as invalid and rejected if they violate one or more rules.

### Kafka Messages

The producer `kafka_producer_case.py` sends raw sales records from the dataset to Kafka.

The Kafka topic for this project is streaming-03-analytics-case.

### Consumer Validation

The consumer `kafka_consumer_p5.py` validates each incoming message against the data contract.

Validation checks include:

- Presence of all required fields
- Reference table lookups for region and product IDs
- Allowed-value checks for categorical fields
- Data type and formal validation for numeric, datatime, and boolean fields

Messages that are accepted are enriched and included in streaming analytics.
They are written to the consumed output CSV file and logged as valid.

Messages that are rejected are written to a rejected output file.
They are writting with detailed validation error messages.

Rejected messages are excluded from analytics to prevent invalid data from affecting results.

Validation protects downstream analytics and ensures reliable and trustworthy outputs.
